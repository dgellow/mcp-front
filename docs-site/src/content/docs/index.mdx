## The Problem

MCP servers use static bearer tokens stored in configuration files. This creates security vulnerabilities:

- **No user attribution** - Can't track who accessed what
- **No revocation** - Can't disable access for ex-employees  
- **Plaintext secrets** - Database credentials exposed
- **No restrictions** - Anyone with the token has full access

## How It Works

<div class="architecture-diagram">
  <object data="/mcp-front/architecture.svg" type="image/svg+xml" aria-label="MCP Front Architecture">
    <img src="/mcp-front/architecture.svg" alt="MCP Front Architecture" />
  </object>

1. Claude connects to MCP Front endpoint
2. User authenticates via OAuth 2.1 (Google by default)
3. MCP Front validates JWT and checks domain
4. Authorized requests proxy to your MCP servers
5. Responses stream back through Server-Sent Events

## Quick Start

```bash
# Clone the repository
git clone https://github.com/dgellow/mcp-front
cd mcp-front

# Copy example config
cp config-oauth.example.json config.json

# Start with Docker Compose
docker-compose up -d
```
